# Завдання 1
Створено граф за допомогою бібліотеки networkX для моделювання реальної мережі метро Відня.
Створений граф візуалізується, виводяться основні характеристики: кількість вершин, кількість ребер, ступінь вершин.

# Завдання 2
DFS йде в глибину, тобто намагається пройти якомога далі по одному шляху, перш ніж повернутися і спробувати інший варіант. В пріоритеті - дослідити всі вузли по одному напрямку, навіть якщо існують коротші шляхи. Втім, перший знайдений шлях не обов’язково оптимальний, тому виходить довший. В цьому випадку результат - 10.50 км.

BFS йде рівнями: спочатку перевіряє всі вузли, що на відстані 1 крок від старту, потім 2 кроки, і так далі.Він завжди знаходить найкоротший шлях, бо не йде в зайві відгалуження. Перший знайдений шлях завжди найкоротший за кількістю кроків, що часто відповідає найкоротшій відстані, особливо якщо всі ребра мають однакову вагу або схожу довжину. У випадку із віденським метро вийшла відстань 7.75 км.

## Висновок

DFS корисний, коли потрібно просто знайти будь-який шлях, не обов’язково найкоротший.
BFS корисний, коли потрібно знайти найкоротший шлях.
